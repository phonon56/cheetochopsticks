This form creates a Salesforce Case record in your Colorado AG Salesforce org using the Salesforce REST API. The narrative is optionally analyzed by the Anthropic Claude API to generate a routing suggestion (mimicking what Salesforce Einstein AI Classification would do in production). In production, OAuth credentials must live on a backend proxy — never in client-side JavaScript.

Salesforce Object Mapping — Case fields used:

// POST /services/data/v58.0/sobjects/Case { "Subject": "[AG Complaint] Consumer Fraud — Jefferson County", "Description": "[Citizen narrative text]", "Type": "Consumer Fraud", // from chip selection or AI routing "Status": "New", "Origin": "Web", "Priority": "Normal", "IsAnonymous__c":false, // custom field "Language__c": "English", // preferred language for follow-up "Location__c": "Denver, CO", // incident location custom field "AIRouting__c": "Consumer Protection Unit" // Einstein / Claude suggestion } // Auth flow (backend proxy — never client-side): POST /services/oauth2/token grant_type=client_credentials &client_id=YOUR_CONNECTED_APP_CLIENT_ID &client_secret=YOUR_SECRET // stays on server → returns { access_token, instance_url } // Case response: { "id": "5001000000AbcDEAAZ", "success": true } // Case Number auto-generated: "00001042" etc.

Recommended backend architecture: Deploy a lightweight Node.js or Python (FastAPI) proxy at /api/complaints/create that accepts the form payload, exchanges client credentials for a Salesforce access token, creates the Case record, and returns the Case Number. The frontend never touches OAuth secrets. For Einstein AI Classification, add a Salesforce Flow that triggers on Case creation to auto-assign Type and OwnerId based on the Description field keywords.

This demo simulates the API call with a realistic response. Set window.SFCONFIG.demoMode = false and configure proxyEndpoint to enable live Salesforce case creation.
