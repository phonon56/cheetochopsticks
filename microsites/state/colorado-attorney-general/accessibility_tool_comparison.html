Is Siteimprove
earning its keep?

Siteimprove was active on every coag.gov page during this audit — yet all 95 findings went undetected, including 37 critical violations. This analysis examines why, what it costs, and what a better stack looks like.

Recommendation

Cancel Siteimprove at renewal. Replace with WAVE browser extension (free) + Pope Tech monitoring (~$1,200/yr) + quarterly Claude DOM audit (~$300–600/yr). Equal automated coverage. Superior semantic coverage. $25,000+ annual savings.

The core finding

This is not a flaw unique to Siteimprove. Every automated accessibility tool — WAVE, axe, Lighthouse, Siteimprove — detects the same 30–40% of WCAG issues. The 95 findings in this audit fall almost entirely in the 60–70% that no automated tool can reach: focus management, keyboard flow, shadow DOM label failures, semantic heading order, JS-driven state changes, misrouted content links, and Spanish language attributes. Siteimprove at $28,000/year provides identical coverage to WAVE at $0/year.

06 — Before Cancelling

One important question to ask first

The caveat

Siteimprove also provides broken link detection, SEO monitoring, content quality scoring, and analytics beyond accessibility scanning. If those features are actively used by the AG's content team, the calculus changes — you'd be replacing multiple tools, not just one. The question to ask internally: Are the non-accessibility features being used? If the subscription is primarily carrying the accessibility scanning function — and the 95-finding gap confirms that function isn't working — then cancellation at renewal is the clear call. If content governance, link monitoring, and SEO are active workflows, consider whether a lower-cost content quality tool (Screaming Frog, ~$200/yr; Semrush, ~$1,200/yr) could replace those functions while eliminating the $28,000 accessibility premium.
