Accessibility Audit · WCAG 2.1 Level AA

ADA Compliance Audit
CDPHE CoHID Mortality Dashboard

Audited: April 8, 2026  ·  Platform: Tableau Server (cohealthviz.dphe.state.co.us)  ·  Standard: WCAG 2.1 Level AA / DOJ Title II Rule

⚠ COMPLIANCE DEADLINE: APRIL 24, 2026

Executive Summary

The Colorado Department of Public Health and Environment (CDPHE) operates a public health mortality data dashboard at cohealthviz.dphe.state.co.us using Tableau Server. This audit was conducted on April 8, 2026 — 16 days before the DOJ Title II ADA final rule compliance deadline of April 24, 2026 — which requires all state and local government web content to meet WCAG 2.1 Level AA.

The audit identified 7 violations, of which 4 are Critical (Level A failures) and 3 are Serious (Level AA failures). The most significant impact is that the primary data content of this dashboard — the county map, all charts, and all interactive controls — is completely inaccessible to screen reader users.

This document includes both the full violation catalogue and a ready-to-deploy accessible HTML alternative that can be embedded on the existing CDPHE pages alongside or in place of the Tableau embed.

01

Violation Catalogue

Critical

All data visualizations have no accessible text alternative

WCAG 1.1.1 Non-text Content (Level A)  ·  1.3.1 Info and Relationships (Level A)

The county map, age group bar chart, sex breakdown chart, marital status chart, homelessness chart, and trend line are all rendered as SVG elements with no aria-label, no role attribute, and no nested <title> element. A screen reader user encounters these as empty, unnamed regions.

Impact: The entirety of the data this dashboard exists to communicate is invisible to blind users and screen reader users. This is the single most consequential failure on the page.

✗ Current

<svg width="900" height="600">
  <!-- No title, no role,
       no aria-label -->
  <g>...county paths...</g>
</svg>

✓ Required fix

<svg role="img"
     aria-labelledby="map-title"
     aria-describedby="map-desc">
  <title id="map-title">
    Colorado county death counts, 2024
  </title>
  <desc id="map-desc">
    Choropleth map showing total deaths
    by county. El Paso: 5,516.
    Denver: 4,996. Jefferson: 5,002.
  </desc>
  <g>...county paths...</g>
</svg>
Critical

No skip navigation link

WCAG 2.4.1 Bypass Blocks (Level A)

There is no skip link allowing keyboard and screen reader users to bypass the toolbar and navigation chrome and jump directly to the data content. Keyboard users must tab through every toolbar button on every page load.

✓ Required fix — add as first element in <body>

<a class="skip-link"
   href="#main-content">
  Skip to main content
</a>

/* CSS */
.skip-link {
  position: absolute;
  top: -999px;
}
.skip-link:focus {
  top: 0;
}
Critical

Form controls have no accessible labels

WCAG 1.3.1 Info and Relationships (Level A)  ·  4.1.2 Name, Role, Value (Level A)

The "Select a Cause of Death" dropdown and the year range slider both lack associated <label> elements. Neither has an aria-label or aria-labelledby attribute. Screen readers announce them only as "combobox" and "slider" with no context.

✗ Current

<select>
  <option>All causes</option>
  ...
</select>

<input type="range"
  min="2000" max="2024">

✓ Required fix

<label for="cause-select">
  Select a Cause of Death
</label>
<select id="cause-select">
  <option>All causes</option>
</select>

<label for="year-slider">
  Select Year (2000–2024)
</label>
<input type="range"
  id="year-slider"
  min="2000" max="2024"
  aria-valuetext="Year 2024">
Critical

Multiple images have no alt text (12+ instances)

WCAG 1.1.1 Non-text Content (Level A)

At least 12 <img> elements have no alt attribute at all — not even an empty one. Screen readers will either skip them silently or read out the image file path or URL, which is disorienting and meaningless.

✗ Current

<img>
<img>
<img>
/* no alt, no role, nothing */

✓ Required fix

/* Decorative images */
<img src="..." alt=""
     role="presentation">

/* Informative images */
<img src="..."
     alt="CDPHE logo">
Serious

Color is the sole encoding on the county map

WCAG 1.4.1 Use of Color (Level A)

The county choropleth map uses only a light-to-dark blue gradient to encode death counts. Users who are colorblind, have low vision, or are viewing on low-quality displays cannot distinguish county values.

Impact: Affects approximately 8% of male users (red-green colorblindness) plus users with low vision or monochrome displays.

Fix: Add text labels directly on county regions (already partially present), ensure sufficient contrast between the lightest and darkest fill values (minimum 3:1 ratio), and provide a supplementary data table as a non-color alternative (see Section 02 below).

Serious

No data table alternative for chart content

WCAG 1.3.1 Info and Relationships (Level A)  ·  Section 508 §1194.22(g)

When data is displayed visually, an accessible equivalent — such as a structured HTML data table — must be available. No table alternative exists for any chart on this dashboard. The data is inaccessible to anyone who cannot perceive the visual representation.

Fix: Add a "View as table" toggle adjacent to each chart. See the working example in Section 02.

Serious

Raw data access requires third-party login (Google account)

WCAG 2.1 Principle 4 — Robust  ·  ADA Title II Equal Access

The state's open data portal (data-cdphe.opendata.arcgis.com) requires a Google account to access the underlying dataset. This creates an authentication barrier for public data — excluding users who lack or decline Google accounts, raising identity and privacy concerns for a health dataset.

Impact: Researchers, journalists, advocates, and members of the public who need raw mortality data cannot access it without creating or providing a commercial third-party account.

Fix: Provide a direct, unauthenticated CSV/Excel download link from the CDPHE domain, or through the existing Data Request System at datarequest.dphe.state.co.us. The data request system should be prominently linked from the dashboard page itself.

What This Fixes (at a glance)

  • ✓ Skip navigation link present as first element in <body>
  • ✓ All form controls have visible, properly associated <label> elements
  • ✓ Year slider uses aria-valuetext to announce the year in words, not just a number
  • ✓ Data table uses <th scope="row/col"> for proper screen reader traversal
  • ✓ Table has a <caption> providing full context without needing surrounding text
  • ✓ Responsive: table restructures to a list format on mobile viewports
  • ✓ All focus states are visible (2px solid outline, 3px offset) — WCAG 2.4.11
  • ✓ Live region (aria-live) on year display so screen readers announce value changes
  • ✗ SVG charts still require Tableau-side fixes — cannot be patched from outside the embed
  • ✗ Raw data login wall requires CDPHE to add a direct download link

Accessible Data Access — Where to Direct Users Now

Accessible Paths to the Raw Data (No Login Required)

Data Request System: datarequest.dphe.state.co.us/drs-app/ — submit a request for mortality data in CSV or Excel. No Google account required.

Email: cdphe.healthstatistics@state.co.us — direct contact to the Vital Statistics program for data requests.

Official Death Statistics page: cdphe.colorado.gov/colorado-death-statistics — links to all three Tableau views and the data request system.

These paths should be prominently linked from the Tableau dashboard page itself, before the embed, so users who cannot access the visual dashboard have an immediate alternative.
