The current form at civilrights.justice.gov/report is a seven-step wizard that displays a visible notice reading: "Your mobile browser is not compatible with this form." The form asks users to self-categorize their complaint before describing what happened, requires them to traverse multiple screens with separate field validations, and — as of April 2026 — fails several WCAG 2.1 Level AA success criteria that the Department's own rulemaking (28 C.F.R. Part 35 Subpart H, effective April 24, 2026) identifies as the benchmark. This redesign re-orders the interaction: describe first, extract second, review before submitting.

Natural-language primary input UX
One textarea replaces the seven-step wizard. Users with cognitive, attention, or memory disabilities do not have to hold field requirements in mind while also recalling the event. Field extraction happens after a complete description is written, not during it.
Mobile-first layout with no browser gatekeeping WCAG 4.1.1
Works on any modern browser including mobile. No "this form is not compatible with your browser" notice. No JavaScript-required hard block. The design tests cleanly on iOS Safari and Android Chrome with assistive-technology pairings.
Every form input has a programmatic label WCAG 1.3.1, 3.3.2, 4.1.2
No inputs rely on placeholder text or visual proximity alone. Screen-reader users hear each field's name and purpose. Required fields use aria-required; errors use role="alert".
Transparent extraction with source quotes Trust
When a field is auto-filled, the source quote from the user's description is shown beneath it. Users can verify exactly why a field was populated. No black-box inference; no AI-sounding confidence they can't interrogate.
Alternate-path cards surfaced before submit Routing
EEOC (employment), HUD (housing), DOT (air travel), Colorado CCRD, Disability Law Colorado, and the ADA Information Line are all linked directly. A user whose issue is workplace discrimination can reach EEOC without completing a DOJ complaint first.
Client-side extraction, no pre-submission tracking Privacy
Field extraction runs in the user's browser. Nothing is transmitted until the user clicks submit. The current civilrights.justice.gov form uses government-hosted infrastructure but relies on cloud storage for its page assets; this prototype demonstrates that extraction can be done locally with zero data leakage before consent.
Libre Baskerville + IBM Plex Mono + Inter Typography
A government civil-rights form should communicate institutional weight. Libre Baskerville (serif) for headlines carries that weight. IBM Plex Mono signals secure, official, structured data. Inter is a highly legible body face. The navy + gold palette derives from DOJ's own visual identity.
Anonymous submission is a primary option, not a buried fallback Accessibility / Trust
Anonymous filing is offered as a first-class checkbox on stage one, with clear explanation of what the user gives up (no status updates, no follow-up). Users reporting powerful institutions — or fearing retaliation — should not have to dig for the anonymous option.
