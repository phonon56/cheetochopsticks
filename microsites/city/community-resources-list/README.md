# Helping Hands — El Paso County Community Resources

A searchable, ranked index of community resources adapted from the
[Pikes Peak Library District Helping Hands guide](https://research.ppld.org/HelpingHands).
~200 resources covering crisis lines, food, shelter, medical, financial
assistance, mental health, addiction, employment, immigration, and more.

## What the page does

- **Plain-English need finder.** Type "kids are hungry" or "place to sleep tonight" — the search expands via a synonym map (`kid` → children/family/youth, `spanish` → latino/immigration, etc.) and ranks resources by editor-curated tag hits + keyword matches + convenience score.
- **Convenience scoring 1–5.** ●●●●● = 24/7 hotline, ● = application required. Sorts fastest help to the top by default.
- **Three orthogonal filters** (need-text, name search, category, minimum-speed) work together. The minimum-speed filter is a *threshold* — picking ●●●○○ shows 3, 4, and 5.
- **Crisis banner pinned to the top** with 911, 988, 211, and TESSA — survives every filter.
- **Cards or rows view** with localStorage persistence.
- **Empty state has a Clear-all-filters button** + 211 fallback.
- **Source PDF preserved** in an iframe with a lazy-load + download fallback.

## When you edit the page

The standalone `index.html` is the canonical source. After editing:

```sh
node microsites/city/community-resources-list/dversion/build.mjs
```

The shared transform regenerates `dversion/dist/{html,css,js}` and
`dversion/preview/index.html`. Don't hand-edit anything in `dversion/dist/`.

## DVersion conventions

- **Namespace class:** root is `<div class="cc-helping-hands">`. Add `cc-helping-hands--themed` for the source's editorial palette (cream paper background, Fraunces display + Public Sans body, forest green + rust accents). Without the modifier, the partial inherits the host site's typography and color and only contributes layout, structure, and component styling.
- **CSS isolation:** the whole stylesheet is wrapped in `@scope (.cc-helping-hands)` by the shared build transform.
- **JS isolation:** the whole script is wrapped in an IIFE rooted at the partial element. `document.*` lookups are intercepted by a `Proxy`, so `#main`, `#needInput`, `#resultsGrid`, etc. cannot collide with the host page.

See [`dversion/README.md`](dversion/README.md) for Drupal and WordPress
integration steps, and [`dversion/REMEDIATION-TUTORIAL.md`](dversion/REMEDIATION-TUTORIAL.md)
for a step-by-step ADA remediation walkthrough.

## Accessibility

The page is the remediation of an audit (see the `.docx` in this
folder). Highlights:

- Crisis hotlines pinned at the top with `role="region"` + `aria-label`
- Skip link to `#main`
- `<form role="search">` wrapping the need finder; Enter submits without page reload
- Real `<label>` associations (visually hidden where the placeholder communicates the same info to sighted users)
- `aria-describedby` ties the search input to the lede ("Tell us what's going on…")
- `aria-live="polite"` on result count and results grid — count + new resource cards are announced
- All filter selects + the text-search input have `aria-label`
- View-toggle uses `aria-pressed` (toggle button pattern) and `role="group"` on the wrapper
- Crisis-line phone numbers are real `tel:` links
- `<button type="button">` everywhere a button doesn't submit a form (chips, view toggle, clear-filters)
- WCAG 2.2 SC 2.4.13 (Focus Appearance) — focus states use 3px outline contrasts ≥ 3:1 against adjacent colors
- Empty-state has a Clear-filters button instead of stranding the user
- 1.36 MB PDF iframe uses `loading="lazy"` and provides a download-link fallback inside `<iframe>` for browsers that can't render PDFs inline
